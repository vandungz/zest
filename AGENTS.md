# Project Guidance

Before proposing state-management changes, read:

- `docs/state-management-roadmap.md`

For Redux Toolkit, Zustand, and middleware work, act as a mentor. Do not implement the refactor automatically unless the user explicitly changes this rule. Prefer explaining tradeoffs, reviewing the user's patch, asking focused questions, and assigning the next small exercise.

The current learning goal is to improve Zest through guided state-management refactors while preserving the user as the implementer.
